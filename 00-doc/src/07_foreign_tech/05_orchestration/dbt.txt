

--------------------------------------------------------------definition-------------------------------------------------------------
dbt （data build tool）是一款流行的开源数据转换工具，能够通过 SQL 实现数据转化，将命令转化为表或者视图，
提升数据分析师的工作效率。dbt主要功能在于转换数据库或数据仓库中的数据，在 E（Extract）、L（Load）、T（Transform） 的流程中，
仅负责转换（transform）的过程。 通过 dbt插件，数据分析师能够通过 SQL 直接建立表单并匹配数据，
而无需关注创建 table 或 view 的过程，并且可以直观地看到数据的流动；同时能够运用 dbt 编写 SQL、进行测试、以及包管理等功能。



---优点
1. 避免写DDL语句
2. 统一管理，基于git开发，利于追踪项目

---缺点
1. git使用门槛
2. 不同数据库不同的语法


---------------------------------------------------------------use-------------------------------------------------------------

1. 元数据加载
在seed文件下放置数据、yml文件（map数据文件字段和类型）
dbt seed: 导入数据模型，数据库可查

2. 创建自定义模型
编写sql：fund_portfolio_stats.sql，（ref宏代表表名，{{ ref('fund_name_em') }}）
dbt run -s fund_portfolio_stats：生成模型
dbt docs generate和dbt docs serve： 在web界面看到模型图
